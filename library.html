Overview this page is the worked example

This file loads nothing inline. It links the compiled variables.css, main.css, and its own library.css bundle exactly as any Astro page, WordPress template, or plain HTML page would. Components read private --_* vars that fall back to semantic variables, which fall back to hard literals — so a component renders bare and upgrades automatically once variables.css is present. Cascade layers keep the whole thing overridable by any host.

@layer variables→ reset→ base→ layout→ component→ page→ override → (host CSS, unlayered, always wins)

How this page is built the real consumption pattern

Two artifacts, nothing more: the <head> that links the globals + this page's bundle, and the page bundle itself — a manifest that pulls in only the primitives and composed blocks this page renders.

1 · The <head> — runtime load order

The first three are identical on every page and cache once. Only the last is unique per page and carries just that page's components.

<!-- globals: cached once, shared by every page -->
<link rel="stylesheet" href="assets/css/variables.min.css">   1 · :root { --ui-* }
<link rel="stylesheet" href="assets/css/main.min.css">        2 · reset + base + layout
<link rel="stylesheet" href="assets/css/theme-midnight.min.css"> 3 · optional theme
<!-- this page's bundle: primitives + components + page chrome -->
<link rel="stylesheet" href="assets/css/library.min.css">     4 · THIS page only

2 · The page bundle — src/pages/library.scss

A manifest. It declares the layer order, @uses only what this page renders, and keeps page-specific styling in @layer page.

// src/pages/library.scss   →   library.css
@use "../_layers";                // canonical @layer order

// only what's NOT already global — button/input/textarea are base
// element styles now (they arrive via main.css), so a page skips them:
@use "../primitives/card";      // the one remaining primitive
@use "../components/hero";      // composed block (molecule)

// page-specific chrome, above `base` so it can tweak elements:
@layer page { /* .sg-* styleguide layout, .hero .btn pill tweak, … */ }

That's the entire contract. The buttons below are real <button>/.btn elements styled by main.css — no inline styles, no framework. Toggle ◐ Light/Midnight (top-right) to watch the theme swap by enabling a single stylesheet.

Anatomy the three-tier fallback that makes it portable

Every themeable property flows through one chain: private var ← semantic variable ← hard literal. Reassign one var to restyle — inline, in a page layer, or a theme — with no selector duplication and no !important. 

/* private var  ←  semantic variable  ←  hard literal fallback */
button, .btn {
  --_bg: var(--ui-color-primary, #c33329);   /* ① renders bare */
  background: var(--_bg);                       /* ② adopts variables */
}
.hero .btn {                              /* @layer page */
  --_radius: 999px;                        /* ③ one var, no new rule */
}
This page links variables.css + main.css + library.css (its own bundle from src/pages/library.scss), plus the optional midnight theme. Recompile src/ to refresh — the CSS is never inlined here.
