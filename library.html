SMASCSSarchitecture library

Overview the perfect, portable SCSS architecture

Every value is a CSS custom property. Components read private --_* vars that fall back to semantic tokens, which fall back to hard literals — so a component renders bare (dropped into an Astro island, a WordPress block, or an Elementor widget with nothing else loaded) and upgrades automatically when tokens or a theme are present. Cascade layers keep the whole thing overridable by any host.

@layer reset→ tokens→ base→ layout→ components→ page→ overrides → (host CSS, unlayered, always wins)

Component anatomy the three-tier fallback that makes it portable

Every themeable property flows through one chain. Drop the component anywhere and it still renders; load tokens and it adopts the brand; load a theme and it re-skins — no selector ever changes.

/* private var  ←  semantic token  ←  hard literal fallback */
.ui-button {
  --_bg: var(--ui-color-primary, #c33329);   /* ① renders bare */
  background: var(--_bg);                       /* ② adopts tokens */
}
.ui-button[data-tone="danger"] {
  --_bg: var(--ui-color-danger, #b91c1c);    /* ③ variant only reassigns the var */
}
